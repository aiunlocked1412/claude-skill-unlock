# Storyteller Template — Structure, Archetypes, Craft

## 3-Act Structure (Save the Cat! Beats)

### Act 1: Setup (25% ของเรื่อง)

| Beat | % | Purpose |
|------|---|---------|
| Opening Image | 0% | Hook + set tone |
| Theme Stated | 5% | ใครบางคนพูด theme (ตอนนั้นยังไม่ชัด) |
| Setup | 1-10% | World + character + everyday |
| Inciting Incident | 10% | เหตุการณ์ที่ทำให้ everyday เปลี่ยน |
| Debate | 10-20% | ตัวเอกลังเล — เข้าไปไหม? |
| Break into 2 | 20-25% | ตัดสินใจเข้าสู่โลกใหม่ |

### Act 2: Confrontation (50%)

| Beat | % | Purpose |
|------|---|---------|
| B Story | 25-30% | Subplot / relationship ที่ช่วย theme |
| Fun & Games | 25-50% | ส่งมอบ promise ของ premise |
| Midpoint | 50% | False victory หรือ false defeat |
| Bad Guys Close In | 50-70% | Obstacle เพิ่มขึ้น |
| All Is Lost | 75% | จุดต่ำสุด |
| Dark Night of Soul | 75-80% | Reflection + realize truth |

### Act 3: Resolution (25%)

| Beat | % | Purpose |
|------|---|---------|
| Break into 3 | 80% | ได้ insight + กลับมาสู้ |
| Finale | 85-99% | Climax — ใช้สิ่งที่เรียนรู้ |
| Final Image | 100% | ตรงข้าม opening (แสดง growth) |

---

## 5-Beat Structure (สำหรับเรื่องสั้น <3,000 คำ)

```
1. Hook (0-15%) — ประโยคแรก + set character/world
2. Setup (15-30%) — stakes + ปัญหาเริ่มก่อตัว
3. Turn (30-60%) — เหตุการณ์ที่เปลี่ยน game
4. Climax (60-85%) — peak tension + ตัดสินใจ
5. Resolution (85-100%) — fallout + theme emerges
```

## Character Development Framework

### The 5 Essential Questions
1. **Want** — สิ่งที่ตัวละครคิดว่าอยากได้ (พาไปเจอ plot)
2. **Need** — สิ่งที่ตัวละครต้องการจริงๆ (ปมที่ต้องแก้)
3. **Wound** — ปมในอดีตที่ทำให้เป็นแบบนี้
4. **Lie** — ความเชื่อผิดที่ตัวละครถือ
5. **Truth** — insight ที่ตัวละครต้องค้นพบ

### Example
```
ตัวเอก: แพรว (35, ผู้บริหารหญิง)

Want: ได้ตำแหน่ง CEO (external goal)
Need: เรียนรู้ว่ารักคือสิ่งที่ให้ ไม่ใช่สิ่งที่ต้องพิสูจน์ (internal)
Wound: พ่อทิ้งตอนอายุ 8
Lie: "ถ้าฉันเก่งพอ คนจะไม่ทิ้งฉัน"
Truth: "คนเราทิ้งกันไม่ใช่เพราะเราไม่ดีพอ — มันคือเรื่องของเขา"

Arc: Workaholic → Vulnerable → Balanced
```

## Character Archetypes (Jung + Campbell)

### Protagonist Types
- **The Hero** — กล้าหาญ ปกป้องคนอื่น (Harry Potter)
- **The Everyman** — คนธรรมดาที่เจอเหตุการณ์พิเศษ (Frodo)
- **The Outlaw** — คนนอกระบบที่ท้าทาย (Jack Sparrow)
- **The Explorer** — นักผจญภัย ค้นหาความหมาย (Indiana Jones)
- **The Caregiver** — ผู้ให้ เสียสละ (Mother figures)

### Antagonist Types
- **The Shadow** — ด้านมืดของตัวเอก (Joker vs Batman)
- **The Corrupter** — เริ่มดี กลายเป็นร้าย (Anakin)
- **The Tyrant** — ผู้มีอำนาจที่กดขี่ (Sauron)
- **The Rival** — เท่าเทียมแต่สวนทาง (Salieri vs Mozart)
- **The Beast** — สัตว์ประหลาด/พลังธรรมชาติ (Jaws)

### Mentor Archetypes
- The Wise Mentor (Gandalf)
- The Reluctant Mentor (Haymitch)
- The Dark Mentor (หลอกให้หลง)

## Dialogue Craft

### Subtext Techniques

**❌ On-the-nose:**
> แม่: "แม่รักลูก แต่แม่เป็นห่วงว่าลูกจะเสียใจ"
> ลูก: "ลูกรู้ว่าแม่รักลูก แต่ลูกอยากเลือกเอง"

**✓ With subtext:**
> แม่เดินไปที่ตู้เย็น "ฝนตกนะ เอาเสื้อแขนยาวไปด้วยไหม"
> ลูกหยิบกุญแจรถ "ไม่เป็นไร"
> แม่: "...เปิด wipers ด้วยนะ"
> ลูกหยุด แล้วหันมอง "แม่..."
> แม่: "ไปเถอะ"

### Character Voice Distinction

**เทคนิค:** ให้ตัวละครแต่ละตัวมี:
- **Vocab** ต่างกัน (ภาษาทางการ / แสลง / ศัพท์เทคนิค)
- **Rhythm** ต่างกัน (ประโยคยาว / สั้น / ขาด ๆ หาย ๆ)
- **Verbal tic** เฉพาะตัว (คำติดปาก)

**ตัวอย่าง:**
- ตัวละคร A (อาจารย์): "หากเราพิจารณาปัญหานี้อย่างรอบคอบแล้ว จะพบว่า..."
- ตัวละคร B (วัยรุ่น): "คือแบบ... มันงงมาก อะ"
- ตัวละคร C (นักลงทุน): "มันไม่ scale ผ่าน"

### Action Beats (แทน "said")

**❌ Flat:**
> "คุณเข้าใจไหม" เขาพูด
> "เข้าใจ" เธอตอบ

**✓ With beats:**
> "คุณเข้าใจไหม" เขาวางแก้วกาแฟที่ดื่มไม่หมด
> เธอมองออกหน้าต่าง "เข้าใจ"

Action beats ทำ 3 อย่างพร้อมกัน:
1. บอกว่าใครพูด
2. Reveal emotion/state
3. Set scene/atmosphere

## Show Don't Tell Examples

| Telling ❌ | Showing ✓ |
|----------|----------|
| เขาโกรธมาก | มือของเขากำแน่น เล็บจิกฝ่ามือจนแดง |
| เธอกลัว | เธอนับขั้นบันไดทีละขั้น — หนึ่ง สอง สาม — เพื่อไม่ให้คิดเรื่องที่รออยู่ข้างบน |
| ห้องหรูหรา | โคมไฟคริสตัลสะท้อนบนพื้นไม้โอ๊ค มีกลิ่นไม้จันทน์ลอยอ่อนๆ |
| เขาจน | เขาเลือกบะหมี่กึ่งสำเร็จรูปรสเดิม รสที่ถูกที่สุด |

## Sensory Balance

ดีที่สุด: **ใช้ทั้ง 5 sense ใน scene สำคัญ**

- **Sight** — ภาพ, สี, แสง
- **Sound** — เสียง, music, silence
- **Smell** — กลิ่น (ทรงพลังมาก — connect กับ memory)
- **Taste** — รสชาติ, ความร้อน
- **Touch** — texture, temperature, pain

**ตัวอย่าง scene ที่สมดุล:**
> ประตูเปิด — เสียงบานพับที่ต้องหยอดน้ำมัน (sound) — กลิ่นกาแฟคั่วใหม่ผสมผสานกับกลิ่นหนังสือเก่า (smell) ลอยมา เธอก้าวเข้าไปในพื้นไม้เย็นใต้ถุงเท้า (touch) แสงสีส้มจากหน้าต่างเลื่อนตกบนโต๊ะไม้เก่า (sight) ป้าเจ้าของร้านส่งแก้วกาแฟมา — รสขมแต่หวานเบาๆ ตรงลิ้น (taste)

## Scene Writing Checklist

- [ ] Conflict ภายในหรือภายนอก?
- [ ] เริ่ม late (ตัด setup ที่ไม่จำเป็น)
- [ ] จบ early (ตัดก่อน resolution เต็ม — ให้ reader อยากอ่านต่อ)
- [ ] POV ชัดเจน (ใคร perspective)
- [ ] Tension rising ตลอด scene
- [ ] Sensory detail อย่างน้อย 2-3 sense
- [ ] Dialogue มี subtext
- [ ] จบ scene ด้วย hook / unanswered question

## Theme Integration

**Theme ≠ Moral**
- Moral = "อย่าโกหก"
- Theme = "ความซื่อสัตย์มีราคาที่บางทีเราไม่อยากจ่าย"

**วิธีฝัง theme:**
1. **ตัวละครรอง** พูด theme ตอนต้น (subtle — reader ยังไม่เก็ต)
2. **Events** ที่ตัวละครเจอ = test ของ theme
3. **Climax** — ตัวเอกตัดสินใจที่ prove/disprove theme
4. **Final image** — แสดง theme แบบ visual

## Genre Conventions (สำคัญ! ต้องเคารพแต่ surprise ได้)

### Mystery
- ต้องมี clue จริง — reader ต้องมีโอกาสแก้คดีเอง
- Red herrings (หลอกให้เดาผิด)
- Killer ต้อง introduce ก่อน — ห้ามเป็นคน random
- Fair play rule — ไม่มีข้อมูลที่ detective รู้คนเดียว

### Romance
- 2 คนต้องเจอกันต้นเรื่อง (meet cute)
- Obstacle ที่ทำให้อยู่ด้วยกันยาก
- Dark moment (คิดว่าจบ)
- Happy ever after (HEA) หรือ happy for now (HFN)

### Horror
- Build dread (ไม่ใช่ jump scare)
- สิ่งที่เรารู้ = น่ากลัวน้อยกว่าสิ่งที่เราไม่รู้
- Monster = metaphor ของอะไรบางอย่าง
- Survivor ≠ unchanged

### Sci-Fi
- Rule ของ world ชัด + consistent
- Tech ต้องมี cost
- ถาม "what if" + explore consequence

## Common Mistakes

1. **Info dump backstory** — เล่าอดีตยาวต้นเรื่อง (fix: reveal ตอนจำเป็น)
2. **Purple prose** — คำหรูเกินจำเป็น (fix: simple words better)
3. **Mary Sue / Gary Stu** — ตัวเอกสมบูรณ์แบบ (fix: ให้ flaw จริง)
4. **Telling emotion** — "เขาเศร้า" (fix: show via action)
5. **Dialogue as info** — ตัวละครพูดสิ่งที่รู้อยู่แล้ว (fix: conflict in dialogue)
6. **Flat secondary characters** — ตัวรอง = function เท่านั้น (fix: give them life outside)
7. **Deus ex machina** — แก้ปัญหาด้วย coincidence (fix: setup ก่อน)
