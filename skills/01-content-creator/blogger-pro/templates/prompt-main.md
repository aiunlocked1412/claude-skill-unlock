# Blog Template — Keyword Research & Outline Patterns

## Keyword Research Framework

### Primary Keyword Selection
เลือก keyword ที่:
- **Volume** ปานกลาง (500-5,000/month) — ไม่แข่งโหด แต่มีคนค้น
- **Intent ชัด** — informational (how, why, what) หรือ transactional (buy, review)
- **Long-tail** ง่ายกว่า short-tail (เช่น "ลดหย่อนภาษีฟรีแลนซ์ 2026" > "ภาษี")

### Keyword Types
| Type | ตัวอย่าง | ใช้เมื่อไร |
|------|---------|-----------|
| Informational | "วิธี...", "ทำไม...", "คืออะไร" | บทความ how-to, อธิบาย |
| Commercial | "best...", "รีวิว...", "เปรียบเทียบ" | บทความเปรียบเทียบ / รีวิว |
| Transactional | "ซื้อ...", "ราคา...", "โปรโมชัน" | Landing page |
| Navigational | ชื่อแบรนด์ + เจาะจง | เจาะเฉพาะ brand |

## Title Formulas (ติด CTR)

### Formula 1: How-to + Year
> "วิธี[ทำอะไร] [ปี] — คู่มือครบสำหรับ[กลุ่มเป้าหมาย]"
> ตัวอย่าง: "วิธีลดหย่อนภาษี 2026 — คู่มือครบสำหรับฟรีแลนซ์"

### Formula 2: Number + Benefit
> "[ตัวเลข] [สิ่งที่] ที่ช่วยให้คุณ[ผลลัพธ์]"
> ตัวอย่าง: "7 แอพจัดการเงินที่ช่วยให้คุณเก็บเงินได้เดือนละ 5,000"

### Formula 3: Contrarian
> "ทำไม [สิ่งที่คนเชื่อ] ถึงผิด (และคุณควรทำอะไรแทน)"
> ตัวอย่าง: "ทำไมการออมแบบเดิมๆ ถึงไม่ได้ผลในปี 2026"

### Formula 4: Case Study
> "ผม[ทำอะไร] [ระยะเวลา] แล้ว[ผลลัพธ์] — นี่คือสิ่งที่เรียนรู้"
> ตัวอย่าง: "ผมใช้ AI ช่วยเขียน blog 90 วัน แล้ว traffic พุ่ง 10x"

### Formula 5: Ultimate Guide
> "[คำค้น] — คู่มือฉบับสมบูรณ์ [ปี]"
> ตัวอย่าง: "SSF vs RMF — คู่มือฉบับสมบูรณ์ 2026"

## Intro Hook Patterns

### Pattern 1: ตัวเลขช็อค
> "ปี 2025 คนไทย 47% ยื่นภาษีผิดพลาด — และส่วนใหญ่เป็นฟรีแลนซ์"

### Pattern 2: Story cold open
> "เดือนที่แล้วเพื่อนผมโทรมาตอน 5 ทุ่ม — 'พี่ ผมลืมยื่นภาษี ต้องทำไง?'"

### Pattern 3: Question hook
> "ถ้าคุณเป็นฟรีแลนซ์แล้วอ่านบทความนี้ สิ้นปีคุณจะประหยัดได้เท่าไร?"

### Pattern 4: Counter-intuitive
> "คนส่วนใหญ่คิดว่าฟรีแลนซ์เสียภาษีแพง — จริงๆ แล้วเสียน้อยกว่ามนุษย์เงินเดือนถ้าทำถูกวิธี"

### Pattern 5: Direct promise
> "อ่านบทความนี้จบ คุณจะรู้วิธีลดหย่อนภาษี 5 แบบที่ฟรีแลนซ์ส่วนใหญ่มองข้าม"

## H2 Subheading Recipe

โครง H2 ที่ engagement สูง:

1. **Problem framing** — "ปัญหา #1 ที่ฟรีแลนซ์เจอ"
2. **Step-by-step** — "ขั้นตอนที่ 1: เตรียมเอกสาร"
3. **Option comparison** — "SSF vs RMF — ใช้ตัวไหน?"
4. **Mistake to avoid** — "3 ข้อผิดพลาดที่ต้องระวัง"
5. **Case / example** — "ตัวอย่างจริง — ฟรีแลนซ์รายได้ 80,000/เดือน"
6. **FAQ integration** — "คำถามที่เจอบ่อย"

## Body Writing Rules

### ย่อหน้าสั้น
- 2-4 บรรทัด ต่อย่อหน้า
- Mobile screen อ่านง่าย

### Structure varieties (สลับใช้)
- Plain paragraph (2-3 ย่อหน้า)
- Bullet list (3-7 items)
- Numbered list (step-by-step)
- Table (comparison)
- Blockquote (tip / warning / quote)
- Code block (ถ้าเทคนิค)

### Sentence variety
- สั้น: "มันใช่."
- กลาง: "ผมลองใช้ 30 วัน แล้วได้ผลดี"
- ยาว: "สิ่งที่น่าสนใจคือ คนส่วนใหญ่ไม่รู้ว่าวิธีนี้สามารถประหยัดได้มากกว่า 20,000 บาท/ปีถ้าทำถูกวิธี"

## Meta Description Formula

```
<Hook 1 ประโยค> + <Keyword ธรรมชาติ> + <Soft CTA>
```

ตัวอย่าง:
> "ฟรีแลนซ์เสียภาษีสูงเกินไป? เรียนรู้วิธีลดหย่อนภาษี 2026 ที่ถูกกฎหมาย พร้อมตัวอย่างจริง คลิกอ่านเลย"

## FAQ Generation

สร้าง FAQ 3-5 ข้อจาก:
1. **Google "People Also Ask"** — ไปดูใน SERP
2. **Reddit / Pantip** — คำถามที่คนถามจริง
3. **Sub-questions** จาก H2 ของคุณเอง

**Format:**
```
Q: <คำถามธรรมชาติ ใช้คำแบบคนถาม>
A: <ตอบตรงประเด็น 2-4 ประโยค — ถ้ายาวกว่านั้นให้ link ไปส่วนอื่นของบทความ>
```

## Internal Linking Strategy

- Link 2-3 บทความเก่าในบทความใหม่
- Anchor text ใช้ keyword ของบทความปลายทาง ไม่ใช่ "คลิกที่นี่"
- วางใน context ธรรมชาติ ไม่ใช่รวบไว้ตอนท้าย

**ตัวอย่าง (ดี):**
> "ก่อนอื่นคุณต้องเข้าใจ [วิธีคำนวณภาษีเงินได้](/blog/calculate-income-tax) ก่อน"

**ตัวอย่าง (ไม่ดี):**
> "อ่านบทความที่เกี่ยวข้อง: [คลิกที่นี่](/blog/xxx)"

## SEO Checklist (Pre-publish)

- [ ] Primary keyword ใน title + H1 + first 100 คำ
- [ ] Primary keyword density 0.5-1.5% (ไม่เกิน 2%)
- [ ] Secondary keywords อยู่ใน H2 อย่างน้อย 2-3 ตัว
- [ ] Meta title 50-60 ตัวอักษร
- [ ] Meta description 150-160 ตัวอักษร + keyword
- [ ] URL slug สั้น มี keyword (kebab-case)
- [ ] Featured image มี alt text พร้อม keyword
- [ ] Internal links 2-3
- [ ] External links 1-2 (authoritative sources)
- [ ] FAQ schema markup (ถ้ามี FAQ)
- [ ] Word count > 1,500 (ถ้าเป็น pillar content > 2,500)
