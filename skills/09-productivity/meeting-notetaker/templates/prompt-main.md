# Prompt Formula — Meeting Note-taker

## Template เก็บ context

```
ชื่อประชุม: <kickoff / weekly / 1:1 / retro / ...>
วันที่ + เวลา: <DD/MM/YY HH:MM>
ระยะเวลา: <30 นาที / 1 ชม. / ...>
รูปแบบ: <online (Zoom/Meet/Teams) / onsite / hybrid>
Chair: <ชื่อ>
Attendees: <ชื่อ + role>
Absent: <ชื่อ (เหตุผล)>
Agenda:
  1. ...
  2. ...
เนื้อหาประชุม / transcript:
<วาง raw text มา ไม่ต้องจัด>
```

## Action Item Recipe

```
สำหรับทุก action ที่ระบุ:
- 1 verb ตอนเริ่ม (ส่ง / ทำ / โทร / นัด / review)
- 1 owner (ห้าม 2 ชื่อ)
- 1 due date (วัน/เดือน — ห้าม ASAP)
- Definition of Done — วัดได้ ตรวจได้
- ช่องทางส่งงาน (Slack, email, Notion, ...)
```

## Decision Log Recipe

```
สำหรับทุก decision:
- WHAT: 1 ประโยคชัด
- WHY: เหตุผล 2-3 ข้อ
- ALTERNATIVES considered: option อื่นที่พิจารณา
- WHO supports / objects: ถ้ามี dissent
- EFFECTIVE: เริ่มเมื่อไหร่
- REVERSIBILITY: one-way door (เปลี่ยนยาก) / two-way door (เปลี่ยนได้)
```

## Follow-up Email Recipe

```
Subject: [สรุป] <ชื่อประชุม> - DD/MM/YY
- Greeting + ขอบคุณ (1 บรรทัด)
- Decision (3-5 ข้อ)
- Action items + owner + due (table)
- Next meeting + agenda
- Deadline ให้แย้งสรุป (48 ชม.)
- Closing สุภาพ
```

## Recurring Meeting Templates

### Weekly Standup (15 นาที)
- 1. Done เมื่อวาน
- 2. Plan วันนี้
- 3. Blocker

### 1:1 (30 นาที)
- 1. Wins สัปดาห์นี้
- 2. Challenges + ขอความช่วยเหลือ
- 3. Career / growth conversation
- 4. Feedback 2-way (manager → IC, IC → manager)

### Retrospective Mad/Sad/Glad (60 นาที)
- 1. Glad — สิ่งที่ดี → ทำต่อ
- 2. Sad — สิ่งที่ไม่ดี → แก้ยังไง
- 3. Mad — สิ่งที่ frustrate → root cause
- 4. Top 3 action items สำหรับ sprint หน้า

## Tips

- **Pre-meeting:** ส่ง agenda ก่อน 24 ชม. เพื่อให้คนเตรียม
- **During:** จด decision สด ๆ — read back ที่ประชุมยืนยัน
- **Post:** ส่งสรุปภายใน 24 ชม. → ความจำคนยังสด
- **Follow-up:** check action item ก่อน meeting ครั้งหน้า ไม่ใช่ตอนเริ่มประชุม
