## Qualification Frameworks

### BANT (Basic — SMB)
- **B**udget: มีงบไหม / กี่บาท / approve โดยใคร
- **A**uthority: คุยกับ decision maker หรือ influencer
- **N**eed: ปัญหาจริงหรือ nice-to-have
- **T**imeline: เมื่อไหร่จะตัดสินใจ

### MEDDIC (Enterprise)
- **M**etrics — KPI ที่ต้องขยับ + เลขปัจจุบัน
- **E**conomic Buyer — ใครเซ็น (ไม่ใช่ user)
- **D**ecision Criteria — must-have vs nice-to-have features
- **D**ecision Process — step + people + timeline
- **I**dentify Pain — true pain (revenue/cost/risk impact)
- **C**hampion — internal advocate (gives intel + sells inside)

เพิ่ม: **C**ompetition (MEDDPICC) — เทียบกับใครบ้าง

### SPIN Selling (Discovery)
- **S**ituation: context ปัจจุบัน (ห้าม >3 question)
- **P**roblem: pain ที่เจอ
- **I**mplication: cost of inaction (ทำไมต้องแก้)
- **N**eed-payoff: ถ้าแก้ได้จะดีแค่ไหน (let prospect sell to themselves)

### Challenger Sale (Insight-driven)
1. **Teach** — give insight prospect ไม่เคยรู้
2. **Tailor** — relate ไป context เขา
3. **Take control** — guide conversation + push gently

## Cold Outreach Sequence (8-touch / 21 วัน)

| # | Day | Channel | Template type |
|---|-----|---------|--------------|
| 1 | D0 | LinkedIn connect (no message) | Warm |
| 2 | D2 | Email #1 — Value-first | Insight + CTA |
| 3 | D4 | LinkedIn DM | Reference Email |
| 4 | D7 | Email #2 — Case study | Social proof |
| 5 | D10 | Call attempt #1 | 30 sec voicemail |
| 6 | D14 | Email #3 — Specific question | Provoke reply |
| 7 | D17 | LINE / WhatsApp (Thai) | Personal touch |
| 8 | D21 | Email #4 — Break-up | Permission to close |

### Benchmark (Thai B2B SaaS 2026)
- LinkedIn connect accept: 30-50%
- Email open: 25-40%
- Email reply: 3-8%
- Cold meeting booked: 1-3%
- Demo → opportunity: 50-70%
- Opportunity → close: 20-35%

## Discovery Questions (12 SPIN)

### Situation (max 3)
1. ทีมคุณ handle <process> ยังไงตอนนี้ครับ?
2. ใช้ tool อะไรอยู่?
3. ทีมที่เกี่ยวข้องมีกี่คน?

### Problem (3-5)
4. อะไรคือ challenge หลักตอนนี้?
5. เกิดบ่อยแค่ไหน?
6. ทีมรู้สึกยังไงกับเรื่องนี้?
7. มีอะไรที่ลองแก้แล้วยังไม่ work?

### Implication (3-5)
8. ถ้าไม่แก้ — 6 เดือนข้างหน้าจะเกิดอะไร?
9. กระทบ revenue/customer/team ยังไง?
10. เคยคำนวณ cost of inaction ไหม?
11. เพื่อนในวงการมีปัญหาเดียวกันไหม?

### Need-payoff (2-3)
12. ถ้าแก้ได้จะส่งผลยังไง?
13. ทีมจะใช้เวลาที่ได้กลับมาทำอะไร?
14. ROI ที่คุ้ม — เลขอะไรต้องเห็น?

## Objection Handling — 10 Standard

| Objection | Reframe Pattern |
|-----------|----------------|
| **"แพง"** | "เข้าใจครับ — ขอเทียบกับ cost of inaction ก่อนนะ ปัจจุบันที่ทีม manual ทำ <X ชม./สัปดาห์> × salary = <THB> /year — versus tool <THB>/year ROI Y%" |
| **"ไม่มีเวลาตอนนี้"** | "เข้าใจครับ — แต่อีก 3 เดือน situation เปลี่ยนยังไง? ถ้าเหมือนเดิม → คุย 15 นาทีตอนนี้น่าจะคุ้มกว่า reschedule 3 รอบ" |
| **"ขอคิดดู"** | "ครับ — ขอถามว่าอะไรที่ยังไม่ชัด? ผมจะได้ส่งข้อมูลตรงประเด็น" |
| **"คุยกับ partner ก่อน"** | "ใครคือ partner ครับ? ผมแนะนำให้ join call ครั้งหน้าเลย — จะได้ตอบคำถามตรงไม่ relay" |
| **"เคยลองแบบนี้แล้วไม่ work"** | "ขอเข้าใจก่อน — ลองตัวไหน? เพราะของเราใช้ approach <unique mechanism> ที่ต่างจาก <competitor> ตรง..." |
| **"ใช้ของคู่แข่งอยู่"** | "เข้าใจ — ส่วนใหญ่ลูกค้า move มาจาก <competitor> เพราะ <specific reason 3 ข้อ>. มี migration support ฟรี + first month free" |
| **"ปีหน้ามาคุยใหม่"** | "ขอถาม — Q4 ปีนี้ plan budget Y26 อยู่แล้วใช่ไหม? ถ้าใช่ — ตอนนี้คือเวลา include เรา ไม่ต้องรอ Q1" |
| **"ส่ง info มาก่อน"** | "ผมส่งให้แน่นอน — แต่ขอ schedule 15 min review หลังคุณอ่านเสร็จได้ไหม? ถ้าไม่เหมาะ — ยกเลิกได้ตลอด" |
| **"บอสไม่อนุมัติ"** | "ใช่ครับ — ผมเข้าใจ ขอช่วยทำ business case ให้บอสไหม? ส่วนใหญ่ใช้ format <X>" |
| **"มีของเยอะอยู่แล้ว"** | "ใช่ — ส่วนใหญ่ลูกค้าเหมือนกัน ของเรา replace <X> ตัว และเพิ่ม <Y> ที่ tool เดิมไม่มี" |

## Pipeline Stages + Probability (B2B SaaS)

| Stage | Probability | Exit criteria |
|-------|------------|--------------|
| Lead | 5% | Capture contact |
| MQL | 10% | Marketing qualified (intent signal) |
| SQL | 25% | BANT met + discovery done |
| Proposal | 40% | Quote sent + understood |
| Negotiation | 65% | Pricing/terms in discussion |
| Verbal yes | 85% | Decision communicated |
| Closed-won | 100% | Contract signed |
| Closed-lost | 0% | Reason captured |

### Pipeline Coverage Math
```
Required pipeline = Quota / Win rate
Coverage ratio = Pipeline value / Quota
Healthy: 3-4x
Stretched: 2-3x
Risky: <2x
```

### Sales Velocity Formula
```
Velocity = (# Opps × AOV × Win rate) / Sales cycle (days)
Higher velocity = healthier business
Improve any of 4 levers
```

## Demo Script Structure (45 min)

```
0-5 min:   Re-confirm goal + agenda
           "Last call we discussed <X>. Today plan to..."

5-10 min:  Discovery refresh
           "ก่อนเริ่ม — มีอะไร update จาก call ก่อนไหม?"

10-30 min: Demo (problem-led, not feature-led)
           "<Pain X> → ขอ show วิธีเรา solve"
           "<Pain Y> → ขอ show ส่วนนี้ครับ"

30-40 min: Q&A + objection
           Pause for questions
           Address concerns

40-45 min: Next step (concrete)
           "Schedule contract review next Tuesday?"
           "Who else needs to see this?"
```

### Demo Don'ts
- ห้าม show ทุก feature (overwhelming)
- ห้าม pitch โดยไม่ tied to discovery pain
- ห้าม end without next step

## Negotiation Tactics

### Pre-negotiation
- BATNA (Best Alternative to Negotiated Agreement)
- Walk-away price (never reveal)
- Multiple variables to trade (price, term, scope, payment)
- Anchor high

### Concession Rules
1. **Never give free** — always trade
   - "If you commit 24-month, I can do 15% off"
2. **Smaller each round** — signal nearing limit
   - Round 1: 10% / Round 2: 5% / Round 3: 2%
3. **Ask for reciprocity** every concession
   - "Can I count on signing by Friday?"

### Closing Techniques
- **Assumptive:** "Wednesday or Friday for kickoff?"
- **Alternative:** "Standard or Premium tier?"
- **Urgency:** "Q4 pricing locks Friday"
- **Summary:** "Recap value + final ask"
- **Puppy dog:** "Try 30 days, refund if not happy"

## Common Mistakes

1. Pitch ก่อน qualify — เสีย 60 นาที
2. Free discount แรก — เสีย margin + signal weak
3. Single-thread (1 contact) — ดีลตายถ้าเขาออก
4. "Just checking in" follow-up — value-less
5. ไม่ next step ทุก meeting — momentum lose
6. Sandbagging deal — bias forecast
7. Skip MEDDIC champion — internal stalled
8. Talk > Listen — 70/30 reversed
