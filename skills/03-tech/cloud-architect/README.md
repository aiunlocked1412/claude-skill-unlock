# Cloud Architect

> AI สถาปนิก Cloud — AWS/GCP/Azure Design, Cost Optimization, HA/DR, Multi-Region, IaC (Terraform/CDK)

## ⚡ ใช้ยังไง

```
/cloud-architect
```

## 🎯 ทำอะไรได้บ้าง

- ✅ Architecture Design (3-tier, microservices, serverless, data platform)
- ✅ Cost Optimization — วิเคราะห์ waste + right-sizing + savings plan
- ✅ HA/DR Design ตาม RTO/RPO ที่กำหนด (Tier 1-4)
- ✅ Multi-Region Setup (active-active / active-passive)
- ✅ IaC Code — Terraform, AWS CDK, Azure Bicep พร้อม module structure
- ✅ Security & Compliance Review (IAM, network, encryption, PDPA/GDPR)
- ✅ Well-Architected Review ครบ 5 pillars

## 💡 Use cases

- Startup อยากออกแบบ AWS infrastructure ตั้งแต่ต้น
- ทีม DevOps อยากลด cloud bill ที่พุ่งสูงขึ้นทุกเดือน
- CTO ต้องการ DR plan สำหรับ fintech / e-commerce
- Engineer เขียน Terraform module ไม่รู้จะ structure อย่างไร
- บริษัทอยากได้ Well-Architected Review ก่อนเปิด production

## 🎓 Tips

- ระบุ **cloud provider** + **scale target** → ได้ architecture ที่ตรง
- บอก **SLA + RTO/RPO** → ได้ HA/DR tier ที่เหมาะสม
- แนบ **AWS Cost Explorer screenshot** → ได้ cost saving ที่จำเพาะ
