## Setup (Phase 1: Baseline)

### 1. Initial login + update
```bash
ssh root@<ip>
apt update && apt upgrade -y
apt install -y ufw fail2ban unattended-upgrades htop ncdu tmux curl git
```

### 2. Create user
```bash
adduser deploy && usermod -aG sudo deploy
# laptop: ssh-copy-id deploy@<ip>
```

### 3. Harden SSH
```ini
# /etc/ssh/sshd_config.d/harden.conf
PermitRootLogin no
PasswordAuthentication no
MaxAuthTries 3
AllowUsers deploy
```
`systemctl restart ssh`

### 4. Firewall
```bash
ufw default deny incoming && ufw default allow outgoing
ufw allow 22,80,443/tcp && ufw enable
```

## Runbook: Incident Response

### Scenario 1: Disk full
```
Detect: alert "Disk > 90%"
Diagnose: du -sh /var/log/* | sort -h
Mitigate: clear old log + docker prune
Verify: df -h shows < 80%
Long-term: setup logrotate
```

### Scenario 2: Service down
```
Detect: Uptime Kuma alert
Diagnose: systemctl status myapp; journalctl -u myapp -n 100
Mitigate: systemctl restart myapp
If still fail: rollback to previous deploy
```

### Scenario 3: SSH brute force spike
```
Detect: fail2ban alert > 100 ban/hour
Diagnose: tail -f /var/log/auth.log; fail2ban-client status sshd
Mitigate: extend bantime to 24h, alert security team
Long-term: change SSH port + IP allowlist
```
